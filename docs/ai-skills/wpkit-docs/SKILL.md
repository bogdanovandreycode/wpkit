# wpkit Docs Skill

Use this when updating README, docs, or AI-facing context.

Read these files first:

- `README.md`
- `docs/AI_CONTEXT.md`

Documentation style:

- Keep docs practical and command-oriented.
- Prefer short sections with exact file and command names.
- Explain behavior in terms of generated artifacts and pipeline steps.
- For AI context, optimize for fast orientation and low token use.

Important docs:

- `docs/PLUGIN_CREATION.md`: plugin config and demo mode.
- `docs/SCAFFOLDS.md`: `make:*` commands.
- `docs/BUILD.md`: `.build.json` and build pipeline.
- `docs/AI_CONTEXT.md`: first-read map for AI assistants.
